# Visual review checklist

- [ ] Correct binding reference ID and path opened.
- [ ] Equivalent viewport and device class recorded.
- [ ] Overall composition and visual hierarchy match.
- [ ] App bar geometry, title, actions, and safe area match.
- [ ] Content grid, spacing, alignment, cards, and forms match.
- [ ] Typography hierarchy, color use, radii, elevation, and icons match.
- [ ] Header, bottom navigation, and center scan action align exactly.
- [ ] Balance visibility and financial component states match.
- [ ] Fixed/scrollable regions and keyboard/focus behavior are correct.
- [ ] Loading, empty, error, restricted, pending, and success states match.
- [ ] RTL mirrors without redesign; mixed-direction content remains safe.
- [ ] Large text (including 200%) reflows accessibly.
- [ ] Screenshot/comparison evidence path is recorded.
- [ ] Every deviation has an approved Design Deviation Record.
- [ ] Final decision is Accept, Accept with documented deviation, Reject for
      visual mismatch, Blocked by contract, Blocked by accessibility, or
      Reference unclear.
