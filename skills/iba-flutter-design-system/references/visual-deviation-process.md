# Visual deviation process

When security, contract, accessibility, localization, platform, or responsive
constraints prevent exact fidelity, stop before changing that region. Complete
the repository Design Deviation Record with exact ID/path/region, approved and
proposed behavior, reason, impacts, alternatives, permanence, approver, status,
and follow-up. Preserve the closest safe appearance. Implementation proceeds
only with the required approval; agents never approve their own deviation.
